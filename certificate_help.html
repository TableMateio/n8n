How to bypass certificate warnings in Safari

Step 1: When you see the warning

Safari will show a warning saying "This Connection Is Not Private"

Step 2: Show Details

Click on "Show Details" to see more information about the certificate issue.

Step 3: Visit Website

Click on "visit this website" link at the bottom of the details section.

Step 4: Confirm in Popup

Click "Visit Website" in the popup dialog to confirm you want to proceed.

Step 5: Enter Password

You might need to enter your Mac password to confirm the security exception.

Option 2: Switch to HTTP Mode

If you continue having issues, you can run N8N in HTTP mode without certificates:

pnpm n8n dev-test --http

This will run N8N on HTTP mode, which doesn't require certificates but is less secure.

Once the certificate is added to your system's trusted certificates, you should be able to access N8N without warnings.
