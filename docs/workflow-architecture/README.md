# n8n Workflow Architecture Documentation

This directory contains comprehensive documentation on our n8n workflow architecture, implementation patterns, and development processes.

## Documents

- [N8N-WORKFLOW-ARCHITECTURE.md](./N8N-WORKFLOW-ARCHITECTURE.md) - Core architecture principles and organization for n8n workflows
- [N8N-IMPLEMENTATION-GUIDE.md](./N8N-IMPLEMENTATION-GUIDE.md) - Practical implementation guidelines for building workflows
- [N8N-NODE-PATTERNS.md](./N8N-NODE-PATTERNS.md) - Common node patterns, configurations, and best practices
- [N8N-DEV-WORKFLOW.md](./N8N-DEV-WORKFLOW.md) - Development workflow for n8n workflow creation and maintenance

## Purpose

This documentation provides:

- A structured approach to n8n workflow development
- Patterns and templates for common workflow scenarios
- Best practices for node configuration and usage
- Development workflows and processes specific to n8n

## When to Use This Documentation

- When designing new workflow architecture
- When implementing workflows using n8n
- When you need examples of common node patterns
- When establishing development processes for workflow creation

## Relationships to Other Documentation

- For high-level system architecture, see the [architecture](../architecture/) directory
- For practical n8n guides and walkthroughs, see the [n8n-guides](../n8n-guides/) directory
- For general development processes, see the [dev-process](../dev-process/) directory

## Using These Documents Together

These documents are designed to be used together:

1. Start with **N8N-WORKFLOW-ARCHITECTURE.md** to understand the overall design principles
2. Use **N8N-IMPLEMENTATION-GUIDE.md** to guide your implementation process
3. Reference **N8N-NODE-PATTERNS.md** when configuring specific nodes
4. Follow the processes in **N8N-DEV-WORKFLOW.md** for development and testing
