# Tax Surplus Recovery System Documentation

This directory contains all documentation for the Tax Surplus Recovery System. The documentation is organized into the following categories:

## Documentation Structure

### [Architecture](./architecture)
High-level system design, architecture principles, and component organization.

### [n8n Guides](./n8n-guides)
Practical guides for working with n8n, including workflow management, data handling, and node patterns.

### [Development Process](./dev-process)
Development workflows, best practices, and processes for contributing to the system.

### [Reference](./reference)
Technical reference material, including credentials management, security considerations, and API documentation.

### [workflow-architecture](./workflow-architecture)
Comprehensive documentation of our n8n workflow architecture, implementation guides, node patterns, and development workflow.

## Key Documents

- [STRATEGY.md](./STRATEGY.md) - Overall implementation strategy and roadmap
- [ARCHITECTURE.md](./architecture/ARCHITECTURE.md) - System architecture and organization
- [DECISION_GUIDE.md](./architecture/DECISION_GUIDE.md) - Framework for making architectural decisions

## Using This Documentation

- **New to the project?** Start with STRATEGY.md and ARCHITECTURE.md
- **Working on n8n workflows?** Check the workflow-architecture folder
- **Need specific n8n guidance?** Look in the n8n-guides directory
- **Working on development?** Reference the dev-process directory

## Contributing to Documentation

When adding new documentation:

1. Place it in the appropriate category directory
2. Update the relevant README.md file
3. Add cross-references to related documents
4. Follow the established formatting conventions
